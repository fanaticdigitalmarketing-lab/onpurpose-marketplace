# Windsurf Development Instructions

Follow /docs/dev-rules.md for all changes

## Core Development Rules

- Never break existing functionality
- Always preserve Idea → Service flow
- No preloaded services
- Only user-generated services allowed
- Maintain mobile-first UI
- Keep performance fast

## Implementation Guidelines

When making changes to the OnPurpose codebase, always reference and follow the rules specified in `/docs/dev-rules.md`. These rules ensure the integrity of the Idea → Service Engine and maintain the core functionality that users depend on.

## Priority Rules

1. **Functionality Preservation**: Never break existing features
2. **Idea Engine Integrity**: Always preserve the Idea → Service → Income flow
3. **User-Generated Content Only**: No preloaded or hardcoded services
4. **Mobile-First Design**: Maintain responsive, mobile-first UI
5. **Performance**: Keep load times and interactions fast

## Code Review Checklist

Before submitting any changes:
- [ ] Existing functionality is preserved
- [ ] Idea → Service flow is intact
- [ ] No preloaded services added
- [ ] Mobile UI is maintained
- [ ] Performance is not degraded
- [ ] Follows all rules in `/docs/dev-rules.md`
